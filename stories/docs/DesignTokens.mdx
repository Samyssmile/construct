# Design Tokens

Design tokens are the single source of truth for every visual decision in Construct. They encode colors, typography, spacing, radii, shadows, and motion into named, reusable values that stay consistent across every component and theme.

---

## Two-Tier Token Pipeline

Construct uses a **two-tier** system: primitives define raw values, and semantic tokens map those values to purpose-driven names.

```
tokens/primitives.json
tokens/semantic.light.json
tokens/semantic.dark.json
tokens/semantic.high-contrast.json
        │
        ▼  npm run build
tokens/tokens.css   ← CSS custom properties
tokens/tokens.json  ← Resolved JSON (for tooling)
tokens/tokens.ts    ← TypeScript exports
```

> **Never hand-edit the generated files.** Edit the source JSON files and run `npm run build`.

### Tier 1 — Primitives

Raw design values that form the palette. They rarely change.

```json
{
  "color": {
    "orange": { "500": "#F4581C" },
    "stone": { "950": "#16130F" }
  },
  "space": { "4": 8, "6": 16 },
  "radius": { "md": 8, "lg": 12 }
}
```

### Tier 2 — Semantic

Contextual aliases that reference primitives by role. Swapping a theme means remapping semantic tokens — primitives stay untouched.

```json
{
  "color": {
    "brand": { "primary": "{color.stone.950}", "accent": "{color.orange.500}" },
    "text":  { "primary": "{color.stone.950}" }
  }
}
```

## Typography Tokens

### Font Families

| CSS Variable | Value |
|-------------|-------|
| `--font-family-brand` | Lato (display & headings) |
| `--font-family-text` | Lato (body) |
| `--font-family-mono` | JetBrains Mono (code & data) |

Lato is a humanist sans-serif — warm, highly legible, with classic proportions — paired with JetBrains Mono for code and data. The aliases `--font-family-heading`, `--font-family-body`, and `--font-family-code` resolve to brand, text, and mono respectively. Lato ships in 400, 700, and 900; the `medium` (500) and `semibold` (600) weight tokens resolve to the nearest available face (400 and 700).

## Using Tokens in CSS

```css
.my-card {
  background: var(--color-bg-surface);
  border: 1px solid var(--color-border-subtle);
  border-radius: var(--radius-card);
  box-shadow: var(--shadow-card);
  padding: var(--space-8);
}

.my-card__title {
  font-family: var(--font-family-heading);
  font-size: var(--font-size-2xl);
  color: var(--color-text-primary);
}
```

## Using Tokens in TypeScript

```ts
import { tokens, cssVars } from '@neuravision/construct/tokens/tokens';

// Resolved hex value
const brand = tokens.semantic.color.brand.primary; // '#16130F'

// CSS variable reference string (for inline styles)
const brandVar = cssVars.semantic.color.brand.primary; // 'var(--color-brand-primary)'

// Access theme-specific overrides
const darkBg = tokens.semanticThemes.dark.color.background.canvas;
```

```bash
npm run build
```

```bash
npm run check
```
