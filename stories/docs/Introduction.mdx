## See it before you read it

Construct ships real, accessible components — not a swatch library. Here are a
few, untouched, in the light theme:

<div className="ct-stage">
  <button className="ct-button">Save changes</button>
  <button className="ct-button ct-button--accent">Publish</button>
  <button className="ct-button ct-button--outline">Cancel</button>
  <label className="ct-switch">
    <input className="ct-switch__input" type="checkbox" defaultChecked />
    <span className="ct-switch__label">High contrast</span>
  </label>
  <label className="ct-check">
    <input className="ct-check__input" type="checkbox" defaultChecked />
    <span className="ct-check__label">Keyboard ready</span>
  </label>
  <span className="ct-chip ct-chip--solid">AA</span>
  <span className="ct-chip">Design tokens</span>
</div>

Notice the orange. Every checked, selected, focused, or current state is marked
by the **same line — the datum.** It is the one decision that makes Construct
legible and recognisable at once — and it survives light, dark, and
high-contrast without dropping below WCAG&nbsp;2.1&nbsp;AA. Open any component
and use the **theme switcher** in the toolbar to watch the contract hold.

## One idea, all the way down

Construct is built on a single conviction: **accessibility and identity are the
same job.** The orange datum that marks every active state is also what makes the
system unmistakably itself — and the warm-concrete neutrals, the ink, and Manrope
are tuned so that contrast is never the price of character.

Pick a component from the sidebar to see it in every theme, read its markup, and
run it through the live accessibility checks built into this Storybook.
